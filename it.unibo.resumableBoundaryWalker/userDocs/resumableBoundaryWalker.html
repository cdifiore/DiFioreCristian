Lab ISS | the project resumableBoundaryWalker

Introduction

This case-study starts to deal with the design and development of proactive/reactive software systems which work under user-control.

Requirements

Design and build a software system (named from now on 'the application') that leads the robot described in VirtualRobot2021.html to walk along the boundary of a empty, rectangular room under user control.
More specifically, the user story can be summarized as follows:
the robot is initially located at the HOME position, as shown in the picture on the rigth
the application presents to the user a consoleGui similar to that shown in the picture on the rigth
when the user hits the button RESUME the robot starts or continue to walk along the boundary, while updating a robot-moves history;
when the user hits the button STOP the robot stop its journey, waiting for another RESUME ;
when the robot reachs its HOME again, the application shows the robot-moves history on the standard output device.

Requirement analysis

dall'interazione con il cliente sono stati chiariti i seguenti termini:
  • Robot: dispositivo in grado di eseguire comandi inviati da un utente utilizzatore, le specifiche dettagliate sono disponibili nel documento VirtualRobot2021.html;
  • Room: una stanza convenzionale (rettangolare) che si può trovare in qualsiasi appartamento;
  • Boundary: confine della stanza, che è solitamente delimitato da delle parte;
  • Walk: movimento del robot, in base alle specifiche riportate in VirtualRobot2021.html esso può muoversi avanti indietro e ruotare a destra e sinistra di novanta gradi;
  • User control: comandi inviati dall'utente che intende comandare il robot per imporre un comportamento;
  • Home: posizione della stanza in cui il robot viene posizionato al suo primo inserimento nella stanza;
  • consoleGui: interfaccia grafica che permette all'utente utilizzatore di inviare o un comando Resume o un comando di Stop;
  • robot-moves history: struttura dati non ancora definita in grado di tenere traccia dei movimenti che questo ad effettuato. quando il robot passa per la posizione home deve essere mostrata su standard output;
  • Resume: comando che impone al robot di iniziare a muoversi o di riprendere il movimento in seguito ad uno Stop;
  • Stop: comando che impone al robot di interrompere il suo movimento.

Problem analysis

Viene sottolineato che:
  1. per il funzionamento dell'applicazione è sufficiente che il robot si muova in avanti e possa effettuare una rotazione di novanta gradi a sinistra;
  2. i comandi invaiti al robot per eseguire il cammino lungo il perimetro della stanza possono essere inviati mediante due protocolli di comunicazione:
    • inviando messaggi alla porta 8090 usando il protocollo HTTP con il metodo POST
    • inviando messaggi alla porta 8091 usando una webSocket
  3. con riferimento al livello tecnologico, ci sono diverse libreria in vari linguaggi di programmazione chiariscesuppportano i protocolli utilizzabili per la comunicazione con il robot.
    • sebbene il problema introduce un'abstraction gap a livello concettuale, non essendoci requisiti che impongono una tipologia di comunicazione preferibile, si opterà per l'utilizzo di protocolli del tipo Request Response o fire and forget.

Logical architecture

si dovrà progettare ed implementare un sistema distribuito formato da due macro componenti:
  1. il VirtualRobot2021 reso disponibile dal cliente
  2. la nostra applicazione resumableBoundaryWalker che può interagire con il robot o tramite protocollo Request-Response sfruttando HTTP-POST o con un protocollo di comunicazione asincrono avvelendosi delle webSocket
